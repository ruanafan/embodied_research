# π0.5

## 1. 研究问题
当直接来自目标域的 robot data 相对有限时，VLA policy 如何泛化到全新家庭中的长时程（long-horizon）、灵巧 household manipulation？来源：Zotero item `NWU9GKBA`，abstract 与 introduction。

## 2. 核心贡献
1. 提出 `π0.5`，一种先预测 semantic subtask、再预测 low-level continuous actions 的分层 VLA。
2. 引入异构联合训练（heterogeneous co-training）配方，混合 robot action data、semantic subtask prediction、verbal supervision、object detection、captioning、question answering 和其他 multimodal web tasks。
3. 使用两阶段训练 pipeline：先用 FAST action tokenization 进行 discrete-token pretraining，再用 flow-matching post-training 实现实时 continuous control。
4. 在新家庭中展示开放世界 household manipulation，并相对 `π0` baselines 提升语言跟随能力。来源：Zotero item `NWU9GKBA`，abstract 与 Sections 1/4。

## 3. 方法概述
模型分两个阶段训练：
1. Pre-training stage：单个 VLA backbone 使用 discrete tokens 在异构机器人和非机器人任务上 co-train，其中 robot action data 由 FAST action tokenization 表示。
2. Post-training stage：模型使用 low-level action examples 和 high-level semantic subtask examples 专门适配移动操作；动作通过 action expert 以 flow matching 建模。

推理时，`π0.5` 先预测高层 subtask，例如 "pick up the plate"，然后以该 subtask 为条件生成 low-level actions。来源：Zotero item `NWU9GKBA`，Figure 3 与 Section 4。

## 4. 实验设置
- 目标域：移动 household manipulation。
- 直接 mobile-manipulator data：约 `400 hours`，采集于真实家庭。
- 数据混合：第一阶段训练中，`97.6%` examples 并非来自目标 mobile household manipulation，而是来自其他机器人和 web-style multimodal data。
- 评估环境包含训练中未见过的三个厨房和三个卧室。
- 定量评估 rubric 在标准 evaluation suites 上报告，附录 protocol 中每个 policy 有 `40` 次 evaluations。来源：Zotero item `NWU9GKBA`，introduction、Figure 7 与 Appendix B/C。

## 5. 关键指标或结果
- 论文称 `π0.5` 在未见家庭中完成了 `items in drawer`、`laundry basket` 和 `dishes in sink` 等任务。来源：Zotero item `NWU9GKBA`，Figure 7 discussion。
- 系统在清理厨房或卧室等例子中执行持续 `10 to 15 minutes` 的长时程多阶段行为，而许多定量 evaluation episodes 仍持续 `2 to 5 minutes`。来源：Zotero item `NWU9GKBA`，abstract 与 Figure 7 discussion。
- 对 language following，附录称 `π0.5` 略优于 `π0-FAST+Flow`，且明显优于 `π0`，说明 discrete-token co-training 对语言理解有价值。来源：Zotero item `NWU9GKBA`，Appendix C/Figure 15。
- 缓存文本确认了这些 gains 的方向，但没有覆盖每个精确 bar-chart number，因此精确 task-wise percentages 仍需直接核验 figures。

## 6. 为什么重要
对 data design 而言，`π0.5` 是一个强例子：广义 VLA 泛化并不只来自扩大 robot demonstrations，而来自混合多层监督：
- low-level motor traces；
- semantic subtasks；
- human verbal guidance；
- 非机器人语义感知数据。

这比单纯 instruction-action imitation 更像一个丰富的 supervision stack。来源：Zotero item `NWU9GKBA`，Sections 1/4 与 Figure 1/3。

## 7. 局限性
- 训练配方规模大、工程负担重，小实验室可能难以复现。
- 许多 open-world performance claims 由图表式 evaluations 支撑，而不是缓存文本中的紧凑 benchmark tables。
- hierarchical split 假设 semantic subtasks 是有用瓶颈；这一点未必能同等迁移到所有 embodiment 或任务族。
- 泛化 claims 很强，但仍集中在作者自己的 data mixture 和 platform stack。
